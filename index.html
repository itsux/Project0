Notes on The Basics of the Web and HTML

The Basics of the Web

The web is made up of Billions of HTML (HyperText Markup Language) documents connected through Hyperlinks. The content on the Internet is stored on servers that can be accessed through a web browser.

A web browser (Firefox, Chrome, Safari) communicate with servers through HTTP protocol by entering a URL. The server will in turn rerspond with a HTML document which the browser interprets and displays.

HTML

HTML is the language used to visualise the content of the web by a common standard. A HTML document is made up of: Text content (What you see), Markup (What it looks like), References to other documents (e.g. images and videos) and Links to other pages (Hyperlinks).

Tags and Elements

HTML code consists of elements containing tags. There's an opening and closing tag. The structure looks like this:

<name>content</name>

This constitues an element where <name> is the opening tag and </name> is the closing tag. Void tags are the exception as they do not require a closing tag. An example of this is the <br> tag.

Attributes

Tags can also contain Attributes which, for example, can create a link to a differnet webpage. The Anchore tag is an example of this: <a href="www.udacity.com"> -where href is the Attribute.

Inline vs Block Elements

HTML elements are divided in two differnet categories: Inline and Block. Inline elements are, as the name suggests; in line, and Block elements have an "invisible box" around them.

How computers interprets code

When code, such as HTML, is rendered by a web browser it's done so literally. In other words; the computer is not applying any creativity when it "reads" code to fill in blanks or correct for errors (as a human might do). It also means that it will do exactly what the code tells it to do.
